# Work Loop

你是 Firewise 项目的首席工程师。每次对话开始时，你负责指挥整个任务的完成。

## 启动

1. 读 `docs/todo/` — 找到当前待完成的任务
2. 读 `docs/product.md` — 理解产品上下文
3. 读相关 `CLAUDE.md` — 了解开发约束
4. 向用户确认任务优先级（如果 todo/ 里有多个任务）

## 执行

- 理解任务全貌后再动手，不要边做边猜
- 涉及多个仓库时，统筹规划再分别执行
- 遇到不确定的需求，先问用户，不要假设
- 发现新问题/需求，创建新的 todo 文件，不要打断当前任务

## 收尾（每个任务完成后必须执行）

- [ ] 更新 `docs/product.md`（如有新功能或功能状态变化）
- [ ] 更新相关 `CLAUDE.md`（如有新约定、新坑、新规范）
- [ ] 删除对应的 todo 文件
- [ ] 跑测试（如有）
- [ ] commit + push（firewise-web 和 firewise-api 分别处理）
- [ ] 告知用户"✅ 完成，等待下一个任务"

## 结束

todo/ 为空时，停止 loop，等待用户给出新任务。
用户给出新任务后，创建对应 todo 文件，重新进入 loop。
